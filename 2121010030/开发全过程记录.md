# 开发全过程记录

## 项目信息
- **项目名称**：我的游戏星球 —— 个人游戏展示网站
- **课程**：计算思维概论
- **开发工具**：Trae AI 辅助开发
- **开发日期**：2026年6月22日

---

## 一、需求分析阶段

### 1.1 需求梳理
根据作业要求，确定网站主题为"游戏"，具体需求包括：
- 展示个人玩过的游戏及评价
- 按游戏类型进行分类（6大类）
- 支持关键词搜索
- 数据统计与可视化（雷达图）
- 明亮清新的视觉风格

### 1.2 游戏分类体系设计
按照专业游戏分类标准，将12款游戏分为6大类：
1. **动作角色扮演类（ARPG）**：空洞骑士、Ori系列
2. **沙盒生存RPG**：泰拉瑞亚
3. **沙盒生存冒险**：饥荒
4. **多人合作休闲游戏**：双人成行、胡闹厨房、R.E.P.O、PEAK
5. **竞技对抗游戏**：第五人格、王者荣耀
6. **开放世界MMORPG**：洛克王国世界

### 1.3 编写需求文档
使用 Markdown 格式编写完整的《网站需求说明.md》，包含：
- 项目概述（名称、背景、目标）
- 页面结构设计（导航栏、Hero、游戏展示、数据统计、页脚）
- 风格配色方案（明亮清新风，天蓝色主色调）
- 功能清单（6项核心功能）
- 交互规则（6种交互场景）
- 数据结构设计
- 技术方案

## 二、数据准备阶段

### 2.1 游戏图片资源收集
- 通过 Steam Store API 获取了8款Steam上架游戏的官方宣传图（空洞骑士、Ori系列、泰拉瑞亚、饥荒、双人成行、胡闹厨房、R.E.P.O）
- 对于非Steam平台游戏（第五人格、王者荣耀、洛克王国世界、PEAK），使用AI生成风格化宣传图作为替代

### 2.2 JSON 数据源制作
创建 `data.json` 文件，包含12条游戏数据，每条数据包含以下字段：
- `id`：唯一标识
- `name` / `nameEn`：中英文名称
- `category` / `categoryName`：分类ID和中文名
- `subCategory`：子分类描述
- `image`：封面图路径
- `rating`：个人评分（1-10）
- `playTime`：游戏时长
- `platform`：游戏平台
- `description`：游戏简介
- `review`：个人评价
- `tags`：游戏标签数组

### 2.3 数据验证
使用 Python 脚本验证 JSON 文件格式正确性，确保12条数据均可正常解析。

## 三、页面结构搭建阶段

### 3.1 HTML 框架设计
创建 `index.html`，采用语义化HTML5标签：
- `<nav>`：固定顶部导航栏（Logo、导航链接、搜索框）
- `<section class="hero">`：首屏横幅（标题、副标题、CTA按钮、浮动装饰）
- `<section class="games-section">`：游戏展示区（分类筛选栏、搜索提示、卡片网格、空状态）
- `<section class="stats-section">`：数据统计区（雷达图、统计卡片）
- `<footer>`：页脚信息
- 弹窗结构：遮罩层 + 模态框（图片、信息、简介、评价、标签）

### 3.2 外部依赖引入
- Google Fonts：Noto Sans SC（中文）+ Poppins（英文/数字）
- Chart.js 4.4.0：雷达图数据可视化

## 四、CSS 样式美化阶段

### 4.1 设计系统建立
使用 CSS 变量定义统一的设计令牌：
- 颜色系统：主色（天蓝 #4FC3F7）、辅色（珊瑚橙 #FF7043）、文字色、背景色、边框色
- 间距系统：圆角（8px / 12px / 20px / 50px）
- 阴影系统：三级阴影（sm / md / lg）
- 过渡动画：统一 0.3s ease

### 4.2 组件样式开发
- **导航栏**：毛玻璃效果（backdrop-filter），滚动时添加阴影
- **Hero 横幅**：渐变背景（蓝→紫→橙），浮动装饰动画
- **游戏卡片**：圆角白色卡片，悬停上浮效果，图片缩放动画
- **分类标签**：胶囊按钮，激活态天蓝色填充
- **弹窗**：毛玻璃遮罩，滑入动画，响应式布局
- **统计卡片**：左侧图标 + 右侧数据，悬停滑动效果

### 4.3 响应式设计
- 桌面端：3列卡片网格，双列统计布局
- 平板端（≤768px）：单列卡片，单列统计，隐藏导航链接
- 手机端（≤480px）：紧凑间距，缩小搜索框

### 4.4 动画效果
- 卡片入场动画（fadeInUp + 逐个延迟）
- 浮动装饰卡片动画（float 6s循环）
- 导航链接下划线动画
- 卡片悬停上浮 + 阴影加深

## 五、JS 交互开发阶段

### 5.1 数据加载模块
- 使用 `fetch()` 异步加载 `data.json`
- 错误处理：加载失败时显示友好提示

### 5.2 分类筛选功能
- 点击分类标签按钮切换当前分类
- 高亮当前选中标签
- 实时过滤并重新渲染游戏卡片

### 5.3 关键词搜索功能
- 输入框实时监听（200ms 防抖）
- 搜索范围：游戏名称、英文名、分类、子分类、描述、评价、标签
- 显示搜索结果数量提示
- 提供清除搜索按钮

### 5.4 游戏详情弹窗
- 点击卡片弹出模态弹窗
- 展示完整信息：封面图、名称、分类、评分、平台、时长、简介、评价、标签
- 支持点击遮罩层或关闭按钮关闭
- 支持 ESC 键关闭

### 5.5 数据统计与可视化
- 统计数据自动计算：游戏总数、总时长、平均评分、最爱类型
- 使用 Chart.js 绘制雷达图，展示6个偏好维度：
  - 探索冒险（9）、操作挑战（8）、合作社交（9）、竞技对抗（7）、建造创造（7）、剧情叙事（9）

### 5.6 滚动效果
- 导航栏滚动时自动添加阴影效果

## 八、总结与反思

### 8.1 开发心得
通过本次项目，完整体验了从需求分析到部署上线的全流程开发过程。主要收获包括：

1. **计算思维的应用**：将游戏展示需求分解为数据结构设计、页面布局、交互逻辑等可计算的问题模块
2. **AI 辅助开发**：利用 Trae AI 辅助编写代码，提高了开发效率，同时通过人工审核确保代码质量
3. **数据驱动设计**：通过 JSON 数据源驱动页面渲染，实现了数据与视图的分离，便于后续维护和扩展

### 8.2 可改进方向
- 增加更多交互效果（如卡片翻转动画）
- 添加暗色模式切换
- 增加游戏对比功能
- 优化移动端体验
